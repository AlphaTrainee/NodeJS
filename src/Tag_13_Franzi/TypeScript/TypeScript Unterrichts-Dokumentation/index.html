TypeScript Grundlagen

Warum TypeScript?

  • Typsicherheit: Fehler werden zur Compile-Zeit erkannt
  • Bessere IDE-Unterstützung: IntelliSense und Auto-Vervollständigung
  • Selbstdokumentierender Code: Typen dienen als Dokumentation
  • Refactoring: Sicheres Umstrukturieren von Code

JavaScript vs. TypeScript:

JavaScript (Fehleranfällig)
function addNumbers(a, b) {
    return a + b;
}

// Laufzeit-Fehler möglich!
console.log(addNumbers("5", 3)); // "53" statt 8
console.log(addNumbers(5));      // NaN
TypeScript (Typsicher)
function addNumbers(a: number, b: number): number {
    return a + b;
}

// Compile-Fehler verhindert Laufzeit-Probleme!
console.log(addNumbers(5, 3));   // 8
// addNumbers("5", 3);           // Compile-Fehler
// addNumbers(5);                // Compile-Fehler

Interfaces

Was sind Interfaces?

Interfaces definieren die Struktur von Objekten und stellen sicher, dass bestimmte Eigenschaften vorhanden sind.

interface Student {
    name: string;
    age: number;
    courses: string[];
    isActive?: boolean; // Optional
}

interface Teacher extends Student {
    subject: string;
    salary: number;
}

function printStudentInfo(student: Student): void {
    console.log(`${student.name} ist ${student.age} Jahre alt`);
    console.log(`Kurse: ${student.courses.join(', ')}`);
}

Classes

TypeScript Classes

Classes bieten objektorientierte Programmierung mit Typsicherheit, Zugriffsmodifikatoren und Vererbung.

class Vehicle {
    protected brand: string;
    private _speed: number = 0;

    constructor(brand: string) {
        this.brand = brand;
    }

    public get speed(): number {
        return this._speed;
    }

    public accelerate(amount: number): void {
        this._speed += amount;
        console.log(`${this.brand} beschleunigt auf ${this._speed} km/h`);
    }
}

class Car extends Vehicle {
    private doors: number;

    constructor(brand: string, doors: number) {
        super(brand);
        this.doors = doors;
    }

    public getInfo(): string {
        return `${this.brand} Auto mit ${this.doors} Türen`;
    }
}

Generics

Was sind Generics?

Generics ermöglichen es, wiederverwendbare Komponenten zu erstellen, die mit verschiedenen Typen arbeiten können.

// Generic Function
function identity<T>(arg: T): T {
    return arg;
}

// Generic Array Function
function getFirst<T>(items: T[]): T | undefined {
    return items[0];
}

// Generic Class
class DataStore<T> {
    private items: T[] = [];

    add(item: T): void {
        this.items.push(item);
    }

    getAll(): T[] {
        return this.items;
    }

    find(predicate: (item: T) => boolean): T | undefined {
        return this.items.find(predicate);
    }
}

// Generic Interface
interface ApiResponse<T> {
    data: T;
    status: number;
    message: string;
}

Union Types

Union Types

Union Types erlauben es einer Variable, einen von mehreren verschiedenen Typen zu haben.

// Union Type
type ID = string | number;
type Status = 'loading' | 'success' | 'error';

interface User {
    id: ID;
    name: string;
    age: number | null;
}

function processId(id: ID): string {
    if (typeof id === 'string') {
        return id.toUpperCase();
    } else {
        return id.toString().padStart(4, '0');
    }
}

function handleStatus(status: Status): void {
    switch (status) {
        case 'loading':
            console.log('⏳ Laden...');
            break;
        case 'success':
            console.log('✅ Erfolgreich!');
            break;
        case 'error':
            console.log('❌ Fehler aufgetreten');
            break;
    }
}

Enums

Enumerationen

Enums ermöglichen es, benannten Konstanten zu definieren und den Code lesbarer zu machen.

// Numeric Enum
enum Direction {
    Up,     // 0
    Down,   // 1
    Left,   // 2
    Right   // 3
}

// String Enum
enum Color {
    Red = '#ff0000',
    Green = '#00ff00',
    Blue = '#0000ff'
}

// Mixed Enum
enum ResponseCode {
    Success = 200,
    NotFound = 404,
    Error = 'ERROR'
}

// Const Enum (optimiert zur Compile-Zeit)
const enum Size {
    Small = 'S',
    Medium = 'M',
    Large = 'L'
}

function move(direction: Direction): void {
    switch (direction) {
        case Direction.Up:
            console.log('🔼 Nach oben bewegen');
            break;
        case Direction.Down:
            console.log('🔽 Nach unten bewegen');
            break;
        case Direction.Left:
            console.log('◀️ Nach links bewegen');
            break;
        case Direction.Right:
            console.log('▶️ Nach rechts bewegen');
            break;
    }
}

Functions

Funktionen in TypeScript

TypeScript bietet erweiterte Funktionalitäten für Funktionen wie Überladungen, optionale Parameter und Typisierung.

// Function mit optionalen Parametern
function greet(name: string, greeting?: string): string {
    return `${greeting || 'Hallo'} ${name}!`;
}

// Function mit Default-Parametern
function createUser(name: string, age: number = 18): User {
    return { id: Math.random(), name, age };
}

// Arrow Function mit Typen
const multiply = (a: number, b: number): number => a * b;

// Function Overloads
function format(value: number): string;
function format(value: string): string;
function format(value: boolean): string;
function format(value: number | string | boolean): string {
    if (typeof value === 'number') {
        return value.toFixed(2);
    } else if (typeof value === 'boolean') {
        return value ? 'Ja' : 'Nein';
    } else {
        return value.toUpperCase();
    }
}

// Higher-Order Function
function createValidator<T>(
    validationFn: (value: T) => boolean
): (value: T) => { isValid: boolean; value: T } {
    return (value: T) => ({
        isValid: validationFn(value),
        value
    });
}

Modules

Module System

TypeScript unterstützt ES6 Module für bessere Code-Organisation und Wiederverwendbarkeit.

// math.ts - Export verschiedener Funktionen
export function add(a: number, b: number): number {
    return a + b;
}

export function subtract(a: number, b: number): number {
    return a - b;
}

export const PI = 3.14159;

// Default Export
export default class Calculator {
    private history: number[] = [];

    calculate(operation: string, a: number, b: number): number {
        let result: number;
        switch (operation) {
            case '+':
                result = add(a, b);
                break;
            case '-':
                result = subtract(a, b);
                break;
            default:
                throw new Error('Unbekannte Operation');
        }
        this.history.push(result);
        return result;
    }

    getHistory(): number[] {
        return [...this.history];
    }
}
